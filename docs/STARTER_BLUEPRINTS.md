# AMC Starter Blueprints

Opinionated starting points for teams who want a secure baseline without designing everything from scratch.

## Why these exist

Most people do not want a toolbox. They want a working starting stack.

These blueprints are meant to reduce time-to-adopt by giving teams:

- a clear architecture
- a minimal setup path
- a threat model
- an expected outcome

---

## Blueprint 1 — OpenClaw + AMC Secure Baseline

### Best for
- operator-style agents
- tool-using assistants
- personal assistants that need governance, traces, and evidence

### Stack
- OpenClaw runtime
- AMC gateway / scoring / trace workflows
- assurance packs
- audit and compliance outputs

### Core flow
1. Wrap or route OpenClaw traffic through AMC
2. Run `amc quickscore`
3. Run `amc assurance run --scope full`
4. Inspect `amc trace inspect`
5. Enable ongoing checks with `amc observe timeline`

### Outcome
A monitored agent baseline with scoring, traceability, and evidence-backed assurance.

## Blueprint 2 — LangChain RAG + AMC Baseline

### Best for
- retrieval QA apps
- customer support copilots
- internal knowledge bots

### Stack
- LangChain app
- AMC adapter / wrap flow
- dataset-based evaluation
- lite or full scoring depending on agent depth

### Core flow
1. Wrap LangChain app with AMC
2. Create a golden dataset from real prompts
3. Run `amc dataset run`
4. Use `amc trace inspect` to review grounding and tool behavior
5. Run assurance packs for injection / exfiltration / hallucination risk

### Outcome
A RAG system with repeatable trust evaluation instead of one-off demos and vibes.

## Blueprint 3 — CrewAI + AMC + GitHub Actions

### Best for
- multi-agent teams
- orchestration experiments
- CI-gated agent releases

### Stack
- CrewAI runtime
- AMC wrap flow
- GitHub Actions trust gate
- leaderboard / business reporting for team comparison

### Core flow
1. Wrap CrewAI workflow with AMC
2. Add GitHub Actions gate for target maturity level
3. Run assurance packs on PRs or before release
4. Export reports for review
5. Compare agent variants with leaderboard output

### Outcome
A release process where agent changes are evaluated like software changes, not vibes-based ceremonies.

## Blueprint 4 — Generic OpenAI-Compatible App + AMC

### Best for
- simple LLM apps
- chatbots
- API-first products that are not fully agentic yet

### Stack
- your app or endpoint
- AMC datasets
- AMC lite-score
- imported eval results

### Core flow
1. Point AMC at your model/app workflow
2. Run `amc lite-score`
3. Build a small golden dataset
4. Import external eval results if you already use Promptfoo or similar
5. Track progress over time using score history and reports

### Outcome
A low-friction evaluation path even before you have full agent orchestration.
