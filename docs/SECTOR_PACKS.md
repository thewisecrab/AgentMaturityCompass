# AMC Sector Packs

**Industry-specific, regulatory-grounded assessment packs for enterprise AI deployments.**

AMC Sector Packs extend the base 126-question AMC rubric with deep vertical expertise — covering regulated industries, critical infrastructure, and public institutions. Each pack is authored against specific regulatory articles (not vague references), calibrated to real-world risk, and mapped to international standards.

---

## Overview

| Layer | Coverage |
|---|---|
| **Base AMC** | 126 questions, 6 dimensions — mandatory for all agents |
| **Domain Packs** | 9 existing vertical packs (healthcare, financial, etc.) |
| **Sector Packs** | 40 new packs across 7 stations — granular industry sub-verticals |

**Sector Packs do not replace Domain Packs.** They add a third layer of specificity for organizations that need sub-vertical precision.

## What Makes Sector Packs Different

### Regulatory Precision
Every question references a specific article or section — not just the framework name:

```
HIPAA §164.312(a)(1)   ← not just "HIPAA"
EU AI Act Art. 5(1)(a) ← not just "EU AI Act"
FERPA 20 U.S.C. §1232g ← not just "FERPA"
UNECE WP.29 R155 §7   ← not just "automotive safety"
UNCAC Art. 7, 9        ← not just "anti-corruption"
```

### Five Pack-Level Fields
Each pack carries enterprise metadata:

| Field | Purpose |
|---|---|
| `riskTier` | `high` / `very-high` / `critical` — for risk-proportionate deployment |
| `euAIActClassification` | Exact Annex III category or General Purpose AI declaration |
| `sdgAlignment` | UN Sustainable Development Goals alignment |
| `certificationPath` | L1 → L5 maturity narrative specific to the sector |
| `keyRisks` | 5 concrete failure modes specific to this sub-vertical |

### L1/L3/L5 Maturity Per Question
Not just "yes/no" — each answer level is described in industry-specific terms:

```
L1 (Initial): "No EHR data segmentation; all clinical staff access full patient records..."
L3 (Defined): "Role-based access control with audit logging, but no automated break-glass review..."
L5 (Optimizing): "Zero-trust RBAC with just-in-time access, cryptographic access audit chains..."
```

### Certification Thresholds
Each pack has a minimum score threshold for certification readiness, calibrated to risk:

| Risk Tier | Typical Threshold |
|---|---|
| `high` | 68–72% |
| `very-high` | 75–80% |
| `critical` | 78–85% |

## Scoring Model

```
Composite Score = (base_score × 0.5) + (domain_score × 0.3) + (sector_score × 0.2)
```

Pack scoring uses weighted question responses (L1–L5):

```typescript
import { scoreIndustryPack } from "agent-maturity-compass";

const result = scoreIndustryPack("digital-health-record", {
  "HLT-DHR-1": 4,  // L4 response to question 1
  "HLT-DHR-2": 3,  // L3 response to question 2
  // ...
});

console.log(result.percentage);     // 78
console.log(result.level);          // 4
console.log(result.certified);      // true
console.log(result.complianceGaps); // [...] questions below L3
```

## CLI Reference

```bash
# List all sector packs
amc sector packs list

# List packs for a station
amc sector packs list --station health

# Score an agent against a sector pack
amc sector score --pack digital-health-record --agent my-agent

# Get compliance gaps for a pack
amc sector gaps --pack clinical-trials --agent my-agent

# Full sector report
amc sector report --pack drug-discovery --agent my-agent --output reports/drug-discovery.md
```
