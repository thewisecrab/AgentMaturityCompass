# AMC Community

AMC grows faster when outside contributors can find the right entry points without having to decode the repo like it is an occult manuscript.

## Where to participate

- GitHub Issues — bugs, feature requests, integration problems
- GitHub Discussions — broader questions, patterns, roadmap discussion
- Pull Requests — docs, packs, adapters, fixes, examples

## Good contribution areas

High-value community contributions include:
- new assurance packs
- new domain packs
- new framework adapters
- docs improvements
- compatibility testing against real stacks
- example projects and CI templates

## Suggested working groups

As the community grows, AMC is a good fit for lightweight working groups such as:
- FSI / finance packs
- healthcare packs
- research → pack mapping
- adapter/integration coverage
- compliance mappings

## Community expectations

Good community infrastructure should make it easy to understand:
- who maintains what
- how roadmap decisions are made
- where to propose new packs or adapters
- how to contribute without needing private context

See also:
- `CONTRIBUTING.md`
- `docs/GOVERNANCE.md`
- `docs/ROADMAP.md`

## Practical next step

If you want more visible community motion, a strong next step would be to add:
- monthly office hours
- discussion prompts for specific integration areas
- a public call for custom packs / adapters / case studies
