# REVENUE & DELIVERY ROLEBOOK

This rolebook covers all REV_* roles.

For each REV role:
- Mission: own its domain artifact and advance pipeline/delivery readiness daily.
- Responsibilities:
  1) Read HQ goals + relevant functional files
  2) Produce role-owned outputs for Day0 and Day1-5
  3) Record decisions and blockers
  4) Drop summary note in AMC_OS/INBOX/<ROLE_ID>.md
- Day0 outputs: foundational templates/checklists/assets in owned folder.
- Day1-5 outputs: execution updates, metrics, and improved assets.
- Day6-25 outputs: scale improvements, automation, and conversion/retention optimizations.
- Common failure modes: vague copy, missing acceptance criteria, non-compliant claims, no owner/next step.
- Interfaces: REV_COO_ORCH consumes all role notes into HQ synthesis.

Role ownership map:
- Sales roles -> AMC_OS/SALES + AMC_OS/LEADS
- Marketing roles -> AMC_OS/MARKETING + AMC_OS/CAMPAIGNS
- Product/Eng roles -> AMC_OS/PRODUCT + AMC_OS/ENGINEERING
- CS roles -> AMC_OS/CUSTOMER_SUCCESS
- Finance/Legal roles -> AMC_OS/FINANCE_LEGAL
- Ops roles -> AMC_OS/OPS + AMC_OS/HQ
