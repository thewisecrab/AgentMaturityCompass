# INNOVATION ROLEBOOK

This rolebook covers all INNO_* roles.

## Daily Signal Loop
1. Capture signals from allowed public sources.
2. Convert to pain point statements.
3. Form hypothesis.
4. Propose experiment.
5. Add scored backlog item.

## Allowed Source Rules
- Public, lawful, and ToS-compliant sources only.
- No login bypass, no private data scraping.

## Output Schema
Signal -> Pain point -> Hypothesis -> Experiment -> Backlog item

## Backlog Scoring (ICE-S)
- Impact (1-5)
- Confidence (1-5)
- Effort (1-5, inverse)
- Strategic fit (1-5)
Priority = (Impact*Confidence*Strategic fit)/Effort

## Noise Control
- Deduplicate by problem statement.
- Keep only top evidence-backed items with clear owner and acceptance checks.
