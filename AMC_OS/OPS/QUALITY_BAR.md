# QUALITY BAR

Deliverables must be client-ready, testable, and evidence-linked.

## Baseline Standards
- Clear owner, version/date, and intended audience.
- Claims are truthful, specific, and compliant (no guaranteed revenue/income claims).
- Assumptions and exclusions are explicit.
- Acceptance criteria are measurable, not subjective.
- Next action and accountable owner are present.

## Mandatory Quality Gate
All core deliverables must pass the acceptance matrix in:
- `AMC_OS/OPS/QUALITY_GATE_CHECKS.md`

### In Scope (minimum)
- Lead list
- Offer
- Landing page
- Proposal
- Roadmap

## Release Rule
A deliverable is approved only when:
1. All critical checks pass.
2. Any major exception has owner + due date.
3. Evidence of QA review is logged.

## QA Record Minimum
- Deliverable name + version/date
- Reviewer name
- Tests run + results
- Decision (Pass / Conditional Pass / Fail)
- Follow-up actions
