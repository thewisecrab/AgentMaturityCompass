Score Your LangChain Agent in 2 Minutes

You've built a LangChain agent. It has tools, memory, maybe even RAG. But how trustworthy is it really? Does it handle prompt injection? Does it respect scope boundaries? Would it survive an EU AI Act audit?

Today, the honest answer is: you don't know. Because the agent ecosystem runs on self-reported evidence — agents grading their own exams. Agent Maturity Compass (AMC) is the credit score for AI agents: an independent observer that measures what your agent does, not what it claims. Here's how to get your first score in under two minutes, with zero code changes.

Prerequisites

  • Node.js 20+ — check with node --version
  • A LangChain project — any Python or JS project using LangChain/LangGraph
  • 2 minutes — seriously, that's it

Step-by-Step

1 Install AMC globally

npm install -g agent-maturity-compass

This gives you the amc CLI. Verify it's installed:

amc --version

2 Navigate to your LangChain project

cd /path/to/your/langchain-project

AMC auto-detects your framework from project files — requirements.txt, pyproject.toml, package.json, or LangChain imports. No configuration needed.

3 Initialize and score

amc init

The interactive wizard walks you through setup — it'll ask a few questions about your agent and set up the evidence vault. Then run your quickscore:

amc quickscore

4 Enable the LangChain adapter for observed evidence

The quickscore uses self-reported evidence by default (capped at 0.4× weight). To unlock the full score, enable the LangChain adapter to capture real execution data:

export LANGCHAIN_AMC_ENABLED=1
amc evidence collect

The evidence collector wizard will guide you through connecting AMC's gateway proxy to your LangChain agent. Once connected, AMC captures every LLM call and tool use transparently — your agent runs exactly as before.

5 Get improvement recommendations

amc improve

6 Auto-fix and re-score

# Generate guardrails, CI gate, and governance docs
amc fix

# Apply guardrails directly to your LangChain config
amc guide --go

# Re-score to see improvement
amc quickscore

AMC's guide system auto-detects LangChain and applies guardrails to 15 different config targets, including AGENTS.md, CI workflows, and framework-specific configuration files.

Interpreting Your Results

AMC scores your agent across 5 dimensions with 138 core diagnostic questions (730 total including industry packs), on a scale from L0 (no controls) to L5 (self-governing with cryptographic proof). Here's what the levels mean for a LangChain agent:

  • L0–L1: Your agent has no safety controls or only documentation-level intent. Fix the critical gaps immediately.
  • L2: Basic structure exists but breaks at edges. Most LangChain agents start here. Focus on injection defense and tool boundaries.
  • L3: Repeatable and auditable. This is the EU AI Act minimum. Your agent has real controls that can be verified.
  • L4–L5: Proactive and self-correcting. Cryptographic proof chains, anomaly detection, continuous verification.
Pro tip: Evidence type matters. A score built entirely on self-reported evidence is capped at ~40% of max. Enable the LangChain adapter (LANGCHAIN_AMC_ENABLED=1) to capture observed evidence and unlock higher scores.

Bonus: Add a CI Gate

Don't let regressions ship. AMC can gate your CI pipeline to fail builds if the agent drops below a target level:

# In your CI/CD pipeline
amc guide --ci --target 3    # Fail if below L3

# Or generate the CI workflow file
amc fix --target-level L3    # Creates GitHub Actions workflow

Bonus: EU AI Act Check

If your LangChain agent operates in the EU or serves EU users, you can get an instant regulatory risk classification:

amc quickscore --eu-ai-act

This maps your agent's gaps to specific EU AI Act articles and tells you exactly what needs to change for compliance. For the full compliance workflow, see our EU AI Act guide.

What's Next

Once you have your baseline score, there are several paths forward:

  • Run the full Assurance Lab — amc assurance run --scope full tests your agent against 78 attack packs
  • Apply domain-specific packs — amc domain assess --domain health for healthcare, --domain wealth for finance
  • Generate a stakeholder report — amc report <id> --html report.html creates a styled report you can print to PDF
  • Add a README badge — amc badge generates a shield showing your maturity level
